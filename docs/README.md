## Documentation Standards

### Naming Convention
- `NN-DOCUMENT-NAME.md` - Core documents (numbered)
- Support documents in root folder

### Format
- Markdown with GitHub Flavored Markdown extensions
- Code blocks with language specification
- Tables for structured data
- Internal links use relative paths

### Updates
When updating documentation:
1. Update the document itself
2. Update CHANGELOG.md
3. Note changes in commit messages
4. Update PROJECT-SUMMARY.md if status changes
